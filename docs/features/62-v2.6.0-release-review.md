# 🔍 Review: v2.6.0 Release

> **Feature**: `62` — v2.6.0 Release
> **Date Completed**: 2026-03-15
> **Lifecycle**: Discuss ✅ → Design ✅ → Plan ✅ → Build ✅ → Ship ✅ → Reflect ✅

---

## Summary

Shipped 3 enhancements in a single build session: `Logger` interface for pluggable logging, `LoadConfig[T]()` for typed config loading, and `GenerateTokenFromString()` for UUID JWT support. 15 files changed, 1068 lines added. No breaking changes. MOD-01 (`authstore`) deferred to v2.7.0.

## What Went Well

- **Clean, focused scope** — 3 independent items with no inter-dependencies allowed parallel-style phased building
- **Architecture-to-code accuracy** — cross-check found only 1 minor issue (`go.mod` indirect marker), zero functional discrepancies
- **`LoadConfig[T]()` test completeness** — 8 tests covering all type conversions, defaults, validation, edge cases
- **JWT refactoring** — extracting `jwtConfig()` before adding the new function eliminated code duplication cleanly
- **Logger interface backward compatibility** — changing `Setup()` return type from `*slog.Logger` to `Logger` didn't break any caller because the starter app discards the return value and the 14 global `slog.*` call sites are unaffected

## What Could Be Improved

- **`go.mod` indirect→direct not caught during build** — should have run `go mod tidy` as part of Phase 2 when adding the `validator` import, not as a post-cross-check fix
- **MOD-01 deferred without explicit tracking** — the discussion doc originally listed 4 items for v2.6.0 but MOD-01 was deferred. Should have updated index.md description earlier to reflect the actual scope
- **No deprecation marker on `GenerateToken(uint)`** — architecture says "mark deprecated" but no `// Deprecated:` comment was added to the function doc. Consider adding it in a follow-up.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Return `Logger` interface from `Setup()` | Enables DI and mocking; callers can still use `slog.Default()` for raw access |
| Don't touch 14 global `slog.*` call sites | Works via `slog.SetDefault()` — changing them would be churn with no benefit |
| `LoadConfig[T]()` validates via `validator.Struct()` | Leverage existing dep (gin already uses it), ~0 extra weight |
| `setField()` handles 6 types + Duration + []string | Covers 95% of real-world config needs without over-engineering |
| Defer MOD-01 to v2.7.0 | Authstore needs its own repo setup + schema design, plus it depends on `GenerateTokenFromString()` which just shipped |
| `jwtConfig()` helper for DRY JWT setup | Both `GenerateToken` and `GenerateTokenFromString` share identical secret/expiry logic |
