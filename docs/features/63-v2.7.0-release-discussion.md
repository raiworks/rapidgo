# 💬 Discussion: v2.7.0 Release

> **Feature**: `63` — v2.7.0 Release
> **Date**: 2026-03-15
> **Status**: COMPLETE

---

## 1. What Are We Building?

A minor release (v2.7.0) delivering 2 additive enhancements to the framework: `UUIDBaseModel` for UUID primary keys and `make:module` scaffold command for domain module organization. No breaking changes. No new dependencies.

## 2. Why?

1. **UUID primary keys** — `BaseModel` only supports `uint` IDs. Projects using UUIDs must define their own base model, losing the convenience of the framework's soft-delete and timestamp support. `GenerateTokenFromString()` shipped in v2.6.0 specifically to enable this.
2. **Domain module scaffold** — the starter app supports domain-organized code but there's no `make:module` command to scaffold the full module directory structure. The existing `make:controller`, `make:model`, `make:service`, `make:provider` commands create individual files but not a complete domain module.

## 3. Origin

Items cataloged in `docs/enhancements-improvements-bugs/index.md`. Cross-checked in `03-rapidgo-next-discussion.md`.

## 4. Scope

### Enhancements
- **ENH-02**: `UUIDBaseModel` in `database/models/base.go` — same fields as `BaseModel` (`CreatedAt`, `UpdatedAt`, `DeletedAt`) but with `ID string` (UUID) primary key. Includes `BeforeCreate` GORM hook to auto-generate UUIDs. Both coexist — users choose which to embed.
- **ENH-07**: `make:module [name]` CLI command in `core/cli/make_scaffold.go` — scaffolds a complete domain module directory: `modules/<name>/models/`, `controllers/`, `services/`, `routes.go`.

### Out of Scope
- **ENH-03**: Advanced auth (refresh tokens, lockout, theft detection) — deferred. This is a `rapidgo-modules/authstore` concern, not core framework. Needs its own lifecycle.
- **MOD-01**: `rapidgo-modules/authstore` — deferred. Requires separate repo setup, schema design, and its own design cycle.
- No changes to JWT functions (already complete)
- No database schema changes in core framework

Dependencies: `github.com/google/uuid v1.3.0` already in `go.mod` ✅.

## 6. Dependencies & Risk

| Risk | Mitigation |
|------|------------|
| GORM UUID performance | UUID strings are indexed fine in PostgreSQL (native UUID type) and MySQL. SQLite uses TEXT. |
| UUID version choice | Use `uuid.NewString()` (v4 random) — standard, no ordering concerns for most use cases |
| Module scaffold structure | Follow existing starter app organization pattern — `models/`, `controllers/`, `services/`, `routes.go` |

## 7. Discussion Resolution

Scope reduced from original v2.7.0 plan (which included ENH-03 + MOD-01). Those are deferred to their own lifecycle in `rapidgo-modules`. This release is focused on 2 clean, additive framework enhancements.
