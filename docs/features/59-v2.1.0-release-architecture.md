## 1. Pagination — Promote + Add Cursor-Based (F-002)

### Current State
Pagination is documented as an app-layer pattern (`app/helpers/`). Not an importable framework function.

### Design

Create `database/pagination.go` with both page-based and cursor-based helpers.

```go
// database/pagination.go

// PageResult holds page-based pagination metadata.
type PageResult struct {
    Page       int   `json:"page"`
    PerPage    int   `json:"per_page"`
    Total      int64 `json:"total"`
    TotalPages int   `json:"total_pages"`
}

// Paginate performs page-based pagination on a GORM query.
// Clamps page to min 1, perPage to 1–100 (default 15).
func Paginate(db *gorm.DB, page, perPage int, dest interface{}) (*PageResult, error)

// CursorResult holds cursor-based pagination metadata.
type CursorResult struct {
    NextCursor string `json:"next_cursor,omitempty"`
    PrevCursor string `json:"prev_cursor,omitempty"`
    PerPage    int    `json:"per_page"`
    HasMore    bool   `json:"has_more"`
}

// CursorPaginate performs cursor-based pagination on a GORM query.
// cursor: base64-encoded value of the order column for the last seen item (empty = start).
// orderCol: column to order/cursor by (default "id"). Must be unique or indexed.
// perPage: items per page (clamped to 1–100, default 15).
// direction: "next" (after cursor) or "prev" (before cursor). Default "next".
func CursorPaginate(db *gorm.DB, cursor string, orderCol string, perPage int, direction string, dest interface{}) (*CursorResult, error)
```

**Cursor encoding**: base64 of the column value. Prevents leaking internal IDs as raw integers and handles non-numeric columns.

**Test file**: `database/pagination_test.go`

## 2. WebSocket Room Cleanup (I-001)

### Current State
`Hub.Handler()` already calls `h.Remove(client)` when `onConnect` returns. But if the connection drops without the handler returning cleanly, there's no heartbeat to detect it.

### Design

Add ping/pong heartbeat to `Hub.Handler()`:

```go
// core/websocket/hub.go — changes to Handler()

// New config option
type HubConfig struct {
    PingInterval time.Duration // default 30s, 0 disables
    PongTimeout  time.Duration // default 10s
}

func NewHub() *Hub                      // existing, uses default config
func NewHubWithConfig(cfg HubConfig) *Hub // new

// Handler() changes:
// - Spawns a goroutine that sends Ping every PingInterval
// - Sets a read deadline of PingInterval + PongTimeout
// - On any error (including deadline exceeded), breaks out → Remove(client) fires
```

Also add lifecycle callbacks:

```go
func (h *Hub) OnJoin(fn func(client *Client, room string))
func (h *Hub) OnLeave(fn func(client *Client, room string))
```

**Backward compatible**: `NewHub()` uses 30s/10s defaults. Existing code works identically.

## 3. Queue Retry with Backoff (I-002)

### Current State
Worker uses flat `RetryDelay` (default 30s) for all retries, set globally in `WorkerConfig`.

### Design

Add per-job backoff configuration:

```go
// core/queue/queue.go — add to Job struct
type Job struct {
    // ... existing fields ...
    BackoffSeconds []uint // e.g. [10, 30, 120] — delay after 1st, 2nd, 3rd failure
}

// core/queue/worker.go — change retry logic
// On failure:
//   if job.BackoffSeconds is set and len > 0:
//     delay = job.BackoffSeconds[min(attempt, len-1)]
//   else:
//     delay = w.config.RetryDelay (existing behavior — backward compatible)
```

Add helper for dispatch:

```go
// core/queue/queue.go
func (d *Dispatcher) DispatchWithBackoff(ctx context.Context, queue, typeName string, payload interface{}, backoff []uint) error
```

**Backward compatible**: existing `Dispatch()` uses flat `RetryDelay`. Only `DispatchWithBackoff()` uses per-job exponential delays.

## 4. Rate Limiter Per-User (I-003)

### Current State
`RateLimitMiddleware()` reads `RATE_LIMIT` env, keys by IP only, memory store only.

### Design

Replace with configurable middleware:

```go
// core/middleware/ratelimit.go

type RateLimitConfig struct {
    Rate   string                    // e.g. "60-M", "100-H". Default from RATE_LIMIT env or "60-M"
    KeyFunc func(c *gin.Context) string // custom key function. Default: IP-based
    // Store is always in-memory (ulule/limiter memory store)
}

func RateLimitMiddleware() gin.HandlerFunc                     // existing — IP-based, env config
func RateLimitWithConfig(cfg RateLimitConfig) gin.HandlerFunc  // new — configurable
```

Usage:
```go
// Per-user rate limiting
r.Use(middleware.RateLimitWithConfig(middleware.RateLimitConfig{
    Rate: "100-D", // 100 per day
    KeyFunc: func(c *gin.Context) string {
        userID, _ := c.Get("user_id")
        return fmt.Sprintf("user:%v", userID)
    },
}))
```

**Backward compatible**: `RateLimitMiddleware()` unchanged.

## 6. Notification System (F-004)

### Current State
Framework has events (pub/sub) and mail separately. No unified notification dispatch.

### Design

New package `core/notification/`:

```go
// core/notification/notification.go

// Channel defines how a notification is delivered.
type Channel interface {
    Send(ctx context.Context, notifiable Notifiable, n Notification) error
}

// Notification is the interface all notifications implement.
type Notification interface {
    Channels() []string   // which channels: "database", "mail"
}

// DatabaseNotification is implemented by notifications sent to the database.
type DatabaseNotification interface {
    ToDatabase(notifiable Notifiable) (DatabaseMessage, error)
}

// MailNotification is implemented by notifications sent via email.
type MailNotification interface {
    ToMail(notifiable Notifiable) (MailMessage, error)
}

// Notifiable is the entity receiving the notification (usually a User).
type Notifiable interface {
    NotifiableID() uint
    NotifiableEmail() string
}

// DatabaseMessage is the data stored in the notifications table.
type DatabaseMessage struct {
    Type    string                 // e.g. "new_follower"
    Data    map[string]interface{} // JSON payload
    ReadAt  *time.Time
}

// MailMessage is the data sent via email.
type MailMessage struct {
    Subject string
    Body    string // HTML
}

// Notifier dispatches notifications through their channels.
type Notifier struct {
    channels map[string]Channel
}

func NewNotifier() *Notifier
func (n *Notifier) RegisterChannel(name string, ch Channel)
func (n *Notifier) Send(ctx context.Context, notifiable Notifiable, notification Notification) error
// Send iterates notification.Channels(), finds the registered Channel,
// calls the appropriate To* method, and dispatches.
```

Built-in channels:

```go
// core/notification/database_channel.go
type DatabaseChannel struct { db *gorm.DB }
func NewDatabaseChannel(db *gorm.DB) *DatabaseChannel

// core/notification/mail_channel.go
type MailChannel struct { mailer *mail.Mailer }
func NewMailChannel(mailer *mail.Mailer) *MailChannel
```

Notification model:

```go
// core/notification/model.go
type NotificationRecord struct {
    ID           uint           `gorm:"primaryKey"`
    NotifiableID uint           `gorm:"index"`
    Type         string
    Data         datatypes.JSON
    ReadAt       *time.Time
    CreatedAt    time.Time
}
```

**Test file**: `core/notification/notification_test.go`
