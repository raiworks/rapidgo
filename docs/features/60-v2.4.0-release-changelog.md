## Changes

### Session 1 — 2026-03-15

**Phase 1 — IMP-02: Consistent `config.Load()`**
- Added `config.Load()` to `NewApp()` in `core/cli/root.go`
- Removed redundant `config.Load()` from `work` command (`core/cli/work.go`)
- Removed redundant `config.Load()` from `schedule:run` command (`core/cli/schedule_run.go`)
- Removed unused `config` import from `schedule_run.go`
- Kept early `config.Load()` in `serve` command (needed before `NewApp()` for `RAPIDGO_MODE`)

**Phase 2 — BUG-02: Fix `serveSingle()` Route Registration**
- Added `applyRoutesForMode(r, application.Container, mode)` call in `serveSingle()` after getting router from container
- `SetRoutes()` callback, templates, static files, and health routes now fire in single-port mode

**Phase 3 — BUG-03: Fix `serveMulti()` Provider Middleware**
- Added `GlobalHandlers()` method to `Router` in `core/router/router.go`
- Updated `serveMulti()` to get the container's router and copy global handlers to per-service routers
- Provider-registered global middleware now applies to all per-service routers

**Phase 4 — IMP-01: Configurable Server Timeouts**
- Added `parseDuration()` and `resolveServerTimeouts()` helpers in `core/cli/serve.go`
- Updated both `serveSingle()` and `serveMulti()` to use env-configurable timeouts
- Env vars: `SERVER_READ_TIMEOUT`, `SERVER_WRITE_TIMEOUT`, `SERVER_IDLE_TIMEOUT`, `SERVER_SHUTDOWN_TIMEOUT`
- Defaults unchanged: 15s, 15s, 60s, 30s

**Phase 5 — FEAT-04: Router `NoRoute()`**
- Added `NoRoute(handlers ...gin.HandlerFunc)` method to `Router` in `core/router/router.go`

**Phase 6 — FEAT-03: Health Endpoint with Version**
- Updated `health.Routes()` to accept optional `version ...string` parameter
- `/health` now includes `"version"` field when provided, backward-compatible when omitted
- Call site in `applyRoutesForMode()` now passes `cli.Version`

**Phase 7 — Tests & Version Bump**
- Added `core/cli/serve_test.go`: 5 tests for `parseDuration` and `resolveServerTimeouts`
- Added to `core/router/router_test.go`: TC-26 `NoRoute`, TC-27 `GlobalHandlers`
- Added to `core/health/health_test.go`: TC-04 version included, TC-05 no version backward-compat, TC-06 readiness unaffected
- Bumped `cli.Version` to `"2.4.0"`
