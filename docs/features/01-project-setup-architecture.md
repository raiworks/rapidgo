## Overview

This feature creates the complete directory scaffold and project files for the RapidGo Framework. No business logic — only structure. The output is a Go module that compiles, runs, and prints a startup banner. Every subsequent feature adds code into this pre-existing structure.

## Component Design

### `cmd/main.go`

**Responsibility**: Application entry point — will eventually bootstrap the framework, but in this feature it only prints a startup banner.
**Package**: `main`
**File**: `cmd/main.go`

```go
package main

import "fmt"

func main() {
    fmt.Println("=================================")
    fmt.Println("  RapidGo Framework")
    fmt.Println("  github.com/RAiWorks/RapidGo")
    fmt.Println("=================================")
    fmt.Println("Ready. Waiting for features...")
}
```

This is intentionally trivial. Feature #05 (Service Container) and #06 (Service Providers) will replace this with the full application bootstrap sequence.

**`routes/api.go`**

## Configuration — `Makefile`

```makefile
.PHONY: build run test clean fmt vet lint

# Build the application binary
build:
	go build -o bin/RapidGo ./cmd/...

# Run the application
run:
	go run ./cmd/...

# Run all tests
test:
	go test ./... -v

# Remove build artifacts
clean:
	rm -rf bin/

# Format code
fmt:
	go fmt ./...

# Run go vet
vet:
	go vet ./...

# Run staticcheck (requires: go install honnef.co/go/tools/cmd/staticcheck@latest)
lint:
	staticcheck ./...
```

## Security Considerations

- **`.env.local` is gitignored** — real secrets never reach version control
- **`.env` contains placeholder values only** — no actual credentials
- **`storage/` artifacts are gitignored** — log files, cache, sessions, uploads don't leak

## Trade-offs & Alternatives

| Approach | Pros | Cons | Verdict |
|---|---|---|---|
| Create all directories upfront | Consistent structure from day one; feature branches stay focused on code | Some directories empty for a long time | ✅ Selected — consistency > minimalism |
| Create directories as needed per feature | Smaller initial commit; no empty folders | Feature branches create folders ad hoc; inconsistent timing; merge conflicts | ❌ Directory sprawl |
| Use `go generate` to scaffold | Automated; repeatable | Over-engineering for a one-time operation | ❌ Unnecessary complexity |
| Monorepo with `internal/` | Standard Go convention for private packages | Blueprint uses `core/` + `app/` separation which is more explicit | ❌ Doesn't match blueprint design |
