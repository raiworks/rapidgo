# 🏗️ Architecture: Framework Rename / Rebrand

> **Feature**: `55` — Framework Rename / Rebrand
> **Discussion**: [`55-framework-rename-discussion.md`](55-framework-rename-discussion.md)
> **Status**: 🟢 FINALIZED
> **Date**: 2026-03-07

---

## Overview

Rename every occurrence of "RGo" to "RapidGo" across the entire codebase — Go module path, imports, CLI commands, configuration defaults, templates, Docker files, and documentation. Add a standard MIT License file with RAi Works copyright. Bump version from `0.1.0` to `0.2.0`. This is a mechanical find-and-replace operation with no logic changes.

---

## Replacement Patterns

Six distinct patterns must be replaced in a specific order to avoid partial matches:

| # | Pattern (Old) | Replacement (New) | Context | Scope |
|---|---|---|---|---|
| 1 | `github.com/RAiWorks/RGo` | `github.com/RAiWorks/RapidGo` | Module path & imports | All `.go` files, `go.mod`, docs code blocks |
| 2 | `RGo Framework` | `RapidGo Framework` | Human-readable name | Docs, comments, templates, CLI output |
| 3 | `RGo` (standalone) | `RapidGo` | Framework name in text | Docs, comments, strings, `.env` values |
| 4 | `"rgo"` | `"rapidgo"` | CLI command name | `core/cli/root.go` |
| 5 | `bin/rgo` | `bin/rapidgo` | Binary output path | `Makefile`, `.gitignore` |
| 6 | `rgo_` | `rapidgo_` | DB name/cache prefix defaults | `.env`, `database/connection.go`, tests |

**Replacement order matters**: Pattern 1 (most specific) must be applied before Pattern 3 (least specific) to avoid double-replacement.

### Patterns NOT replaced

| Pattern | Reason |
|---|---|
| `RGO_TEST_VAR`, `RGO_TEST_MISSING` | Test-only env var names in `app/helpers/helpers_test.go` — not framework branding |
| `go.sum` entries | Auto-generated file — will be regenerated by `go mod tidy` |
| Git history | Old commits naturally reference "RGo" — no history rewriting |
| Feature doc filenames | Keep existing names (e.g., `01-project-setup-discussion.md`) — only content changes |
| `RGO_MODE` references in docs | These are in the service-mode discussion doc (#56) — will be renamed to `RAPIDGO_MODE` when that feature is implemented. For now, update the docs to reflect the new prefix |

## Trade-offs & Alternatives

| Approach | Pros | Cons | Verdict |
|---|---|---|---|
| **Full rename (chosen)** | Clean slate, consistent identity, no legacy confusion | ~110 files touched, brief instability during rename | ✅ Selected |
| **Alias only** (keep old imports, add new) | Backward compatible | Confusing dual identity, technical debt | ❌ Half-measures cause more problems |
| **New repository** (fork) | Clean git history | Lose all commit history, two repos to manage | ❌ History is valuable |

## Implementation Strategy

The rename is executed via automated find-and-replace in a specific order, then validated by compilation and tests. Manual intervention is only needed for edge cases.

### Replacement Execution Order

1. **`go.mod`** — Module path (must be first — all imports depend on this)
2. **All `.go` files** — Import paths (`github.com/RAiWorks/RGo` → `github.com/RAiWorks/RapidGo`)
3. **All `.go` files** — String literals and comments (`RGo` → `RapidGo`, `rgo` → `rapidgo`)
4. **`core/cli/root.go`** — Version bump `0.1.0` → `0.2.0`
5. **`.env`** — Configuration defaults
6. **`Makefile`** — Binary name
7. **`Dockerfile`** — Comment (binary stays as `server`)
8. **`Caddyfile`** — Comment
9. **`resources/views/home.html`** — Template branding
10. **All `.md` files** — Documentation references
11. **Create `LICENSE`** — MIT with RAi Works copyright
12. **`go mod tidy`** — Regenerate `go.sum`
13. **`go build ./...`** — Verify compilation
14. **`go test ./...`** — Verify all tests pass

### Validation Gates

| Gate | Command | Expected |
|---|---|---|
| Compilation | `go build ./...` | Zero errors |
| Tests | `go test ./...` | All pass |
| CLI verify | `./bin/rapidgo version` | `RapidGo Framework v0.2.0` |
| CLI serve banner | `./bin/rapidgo serve` | Shows `github.com/RAiWorks/RapidGo` |
| No stale references | `grep -r "RGo" --include="*.go" \| grep -v RapidGo \| grep -v RGO_TEST` | Zero results (except `RGO_TEST_*`) |
