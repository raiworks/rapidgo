# Localization / i18n

## Abstract

This document covers the internationalization system — loading JSON
translation files, the translator API, template interpolation, and
fallback locale support.

## 1. Terminology

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT",
"SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this
document are to be interpreted as described in [RFC 2119].

- **Locale** — A language/region identifier (e.g., `en`, `es`,
  `fr`).
- **Fallback** — The locale used when a translation is missing in
  the requested locale.

## 2. Translation Files

Place JSON files in `resources/lang/`, one per locale:

### `resources/lang/en.json`

```json
{
    "welcome": "Welcome, {{.Name}}!",
    "errors.not_found": "Resource not found",
    "errors.unauthorized": "You must be logged in"
}
```

### `resources/lang/es.json`

```json
{
    "welcome": "¡Bienvenido, {{.Name}}!",
    "errors.not_found": "Recurso no encontrado",
    "errors.unauthorized": "Debes iniciar sesión"
}
```

Keys use dot notation for namespacing.

## 3. Translator

```go
package i18n

import (
    "bytes"
    "encoding/json"
    "os"
    "sync"
    "text/template"
)

type Translator struct {
    mu       sync.RWMutex
    messages map[string]map[string]string // locale → key → message
    fallback string
}

func NewTranslator(fallback string) *Translator {
    return &Translator{
        messages: make(map[string]map[string]string),
        fallback: fallback,
    }
}

func (t *Translator) LoadFile(locale, path string) error {
    data, err := os.ReadFile(path)
    if err != nil {
        return err
    }
    var msgs map[string]string
    if err := json.Unmarshal(data, &msgs); err != nil {
        return err
    }
    t.mu.Lock()
    defer t.mu.Unlock()
    t.messages[locale] = msgs
    return nil
}

func (t *Translator) Get(locale, key string, args ...interface{}) string {
    t.mu.RLock()
    defer t.mu.RUnlock()

    msg := t.resolve(locale, key)
    if msg == "" {
        msg = t.resolve(t.fallback, key)
    }
    if msg == "" {
        return key // return the key itself as fallback
    }

    if len(args) > 0 {
        tmpl, err := template.New("").Parse(msg)
        if err != nil {
            return msg
        }
        var buf bytes.Buffer
        tmpl.Execute(&buf, args[0])
        return buf.String()
    }
    return msg
}

func (t *Translator) resolve(locale, key string) string {
    if msgs, ok := t.messages[locale]; ok {
        if val, ok := msgs[key]; ok {
            return val
        }
    }
    return ""
}
```

### Key behavior:
1. Look up the key in the requested locale.
2. If not found, look up in the fallback locale.
3. If still not found, return the key string itself.

## 4. Usage

```go
trans := i18n.NewTranslator("en")
trans.LoadFile("en", "resources/lang/en.json")
trans.LoadFile("es", "resources/lang/es.json")

msg := trans.Get("es", "welcome", map[string]string{"Name": "Carlos"})
// "¡Bienvenido, Carlos!"

msg := trans.Get("es", "errors.not_found")
// "Recurso no encontrado"

msg := trans.Get("fr", "welcome", map[string]string{"Name": "Pierre"})
// Falls back to English: "Welcome, Pierre!"
```

## 5. Template Integration

Register the translator as a template function:

```go
r.SetFuncMap(template.FuncMap{
    "t":     func(key string) string { return trans.Get(currentLocale, key) },
    "route": router.Route,
})
```

Use in templates:

```html
<h1>{{t "welcome"}}</h1>
<p>{{t "errors.not_found"}}</p>
```

## 6. Security Considerations

- Translation files **SHOULD** be validated at load time to catch
  malformed JSON.
- User-supplied locale strings **MUST** be validated against known
  locales to prevent path traversal (e.g., `../../etc/passwd`).
- Template interpolation in translations uses Go's `text/template`,
  not `html/template` — ensure translated strings don't contain
  unsanitized HTML when rendered in web pages.
